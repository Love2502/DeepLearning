\begin{abstract}
In this project, we worked on image classification with the Oxford-IIIT Pet dataset. The assignment asks for two tasks on the same dataset and a comparison between two model approaches. We solve 37-class breed classification and binary cat-versus-dog species classification. Model 1 is a convolutional neural network trained from scratch. Model 2 uses transfer learning with a pretrained ResNet18 backbone. We compare the models using accuracy, macro F1-score, training curves, confusion matrices, and the design choices from the course.
\end{abstract}

\section{Problem Statement}
Our goal is to build and compare deep learning models for visual pet recognition. The same dataset is used for two related classification tasks:
\begin{enumerate}[label=\arabic*.]
    \item Breed classification with 37 classes.
    \item Species classification with two classes: cat and dog.
\end{enumerate}
These tasks are suitable for the course because they require tensors, datasets and dataloaders, convolutional layers, activation functions, cross-entropy loss, optimizers, validation, and model comparison.

\section{Connection to the Course Material}
Before finalizing the project, we checked the lecture chapters and the provided PyTorch notebooks. The project uses the same ideas in one complete image-classification workflow. Tensors and dataloaders are used for image batches, convolutional layers are used for local feature extraction, ReLU is used as the activation function, cross-entropy is used as the classification loss, and AdamW is used as the optimizer. We also used dropout, data augmentation, validation splits, confusion matrices, and macro F1 because these topics connect directly to the chapters on regularization, loss functions, optimizers, and evaluation.

\section{Dataset Description}
The Oxford-IIIT Pet dataset contains 37 pet categories with roughly 200 images per class. Images vary in scale, pose, and lighting, making the problem more realistic than a simple classroom dataset. The official dataset provides breed labels, binary cat/dog labels, head region annotations, and trimap segmentation annotations. This project uses the category and binary-category targets through Torchvision because they match the required ``multi-class classification + binary classification'' project option.

\section{Preprocessing and Splits}
Images are resized and cropped before being passed into the models. The project defaults use a laptop-friendly resolution of $160 \times 160$. For the final run on our laptop, we used $96 \times 96$ images so that all four required experiments could finish in a reasonable time. Training uses random resized crops and horizontal flips for data augmentation. Validation and test images use deterministic resizing and center cropping. Pixel values are normalized with ImageNet mean and standard deviation, which is especially appropriate for the transfer learning model.

The official \texttt{trainval} split is divided into training and validation subsets with a fixed random seed. The official test split is kept separate for final evaluation. Reproducibility is supported through fixed seeds, scripted experiments, and saved metrics.

\section{Model 1: CNN from Scratch}
The from-scratch model is a compact CNN. It stacks convolutional blocks with:
\begin{itemize}
    \item \texttt{Conv2d} layers for local image feature extraction,
    \item batch normalization for stable training,
    \item ReLU activation functions for non-linearity,
    \item max pooling for spatial downsampling,
    \item adaptive average pooling before the classifier,
    \item dropout to reduce overfitting.
\end{itemize}
The output layer size changes according to the task: 37 logits for breed classification and 2 logits for species classification. Cross-entropy loss is used because both tasks are classification problems with mutually exclusive labels. AdamW is used as the default optimizer because it trains reliably with mini-batches and includes weight decay regularization.

\section{Model 2: Transfer Learning}
The alternative model uses ResNet18 pretrained on ImageNet. The convolutional backbone is frozen first, and only the new classifier head is trained. The final ResNet block is then unfrozen for a short fine-tuning phase. Transfer learning is expected to perform well because early CNN layers learn generic visual features such as edges, textures, and shapes.

\section{Comparison and Discussion}
The scratch CNN directly tests whether a model can learn useful visual features from the Oxford Pets training data alone. It is transparent and closely connected to the course topics, but it has limited prior knowledge and may require more data or training time.

The transfer learning model starts with strong general image features from ImageNet. In the laptop run it clearly outperformed the scratch CNN on both tasks. For breed classification, transfer learning improved test accuracy from 0.049 to 0.308. For species classification, transfer learning improved test accuracy from 0.685 to 0.898 and macro F1 from 0.529 to 0.877. This supports the expectation that pretrained convolutional features transfer well to natural-image tasks.

The difference between the two tasks is also important. Species classification is easier because the model only needs to separate cats from dogs. Breed classification is harder because it contains 37 classes, some breeds are visually similar, and the laptop run uses fewer examples per class. The comparison is useful for the course because the scratch model makes convolution, activation, loss, optimizer behavior, and regularization easy to explain, while the transfer model shows how pretrained features can improve performance in practice.

\section{Limitations}
The project uses moderate image resolution and short training runs so all students can run it on a laptop. The reported final results use stratified subsets rather than the full dataset, which makes the project reproducible in class but reduces the maximum possible accuracy. A longer full-dataset run with larger images would likely improve both models, especially the scratch CNN. Future work could include segmentation from the trimap annotations, larger image sizes, longer fine-tuning, or model interpretation methods such as Grad-CAM.
